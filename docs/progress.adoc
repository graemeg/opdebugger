[[Embedding Decision]]
=== Embedding Decision: External File vs ELF Section (RESOLVED)

==== Problem

The FPC `TDebugInfo` infrastructure is designed around emitting debug data into asm lists that flow through the assembler and linker. All existing debug formats (DWARF, Stabs, CodeView) work this way. The external .opdf file approach fights this architecture and makes line information impossible without a post-link fixup step.

==== Decision

Pivot to embedding OPDF data inside the binary as a custom ELF section (`.opdf`). This aligns with FPC's debug writer infrastructure and enables:

* Label-based address resolution through the assembler/linker
* Line information support (critical for a working debugger)
* Single-file distribution (no separate .opdf file to manage)
* The OPDF specification already planned for embedded storage

==== Implementation (completed in Phase 1F)

* [x] Define OPDF ELF section name and structure
* [x] Register a new asm list for OPDF data emission
* [x] Rewrite TOPDFDebugWriter to emit tai_const values into asm list
* [x] Update PDR debugger to read OPDF section from ELF binary
* [x] Implement insertlineinfo with label insertion and line records
* [ ] Update OPDF specification to document embedded storage format
* [ ] Update test infrastructure to work with embedded OPDF

== Quick Reference

=== Build Commands

[source,bash]
----
# Build all modules (standalone debugger)
cd /data/devel/opdebugger
pasbuild compile

# Build FPC compiler with OPDF support
cd /data/devel/fpc-3.3.1/src
make -C compiler cycle
----

=== Testing

[source,bash]
----
# Compile test with FPC OPDF support (use ppcx64 directly, not fpc driver)
/data/devel/fpc-3.3.1/src/compiler/ppcx64 -gO test_01_primitives.pas

# Compile test with DWARF + opdf_generator (current working approach)
cd tests/integration
fpc -g test_01_primitives.pas
../../tools/target/opdf_generator test_01_primitives

# Run debugger
../../pdr-cli/target/pdr test_01_primitives

# Run automated test suite
bash run_launch_tests.sh
bash run_launch_tests.sh test_01_primitives  # single test
----

=== Debugger Commands

[source]
----
run                          Start program (pauses at entry)
break file.pas:22            Set breakpoint by file:line
break 0x401000               Set breakpoint by hex address
break MyVar                  Set breakpoint by variable name
continue / c                 Resume execution
next / n                     Step to next source line
step / s                     Single instruction step
print <var>                  Print variable value
callstack / cs               Show call stack
delete <num>                 Remove breakpoint
quit                         Exit debugger
----
