Phase 2: FPC OPDF Type Handlers Implementation Plan

 Context

 All FPC integration phases 1A-1G are complete. The compiler emits OPDF data into an .opdf ELF section with primitive type records
 (recPrimitive), global variable records (recGlobalVar), and line info records (recLineInfo). The standalone debugger successfully reads these
  from embedded ELF sections.

 Phase 2 implements the remaining appenddef_* and appendsym_* handlers in dbgopdf.pas so the FPC compiler emits complete type and symbol
 information matching the OPDF format defined in ogopdf.pas.

 Additionally, a new recInterface record type (value 16) will be added to the OPDF specification to properly support COM and CORBA interface
 debugging — addressing the known FPC limitation where DWARF info lacks interface method details (FPC issue #39904).

 Files to Modify

 - /data/devel/fpc-3.3.1/src/compiler/dbgopdf.pas — All type and symbol emission handlers (primary file)
 - /data/devel/opdebugger/opdf-lib/src/main/pascal/ogopdf.pas — Add recInterface record type definition
 - /data/devel/opdebugger/opdf-lib/src/main/pascal/opdf_io.pas — Add interface reader/writer methods
 - /data/devel/opdebugger/docs/progress.adoc — Update progress tracker

 Float types reuse REC_PRIMITIVE (same format: TypeID + size + signedness + name). Property and field records are embedded inline within
 class/record records, not emitted separately.

 Implementation Steps (ordered by dependency)

 Step 1: Add record type constants to dbgopdf.pas

 Add all missing REC_* constants to the existing const block.

 Step 2: appenddef_float — Float types as primitives

 Emit REC_PRIMITIVE records for float types. Extract from TFloatDef:
 - def.floattype — determines the float kind
 - def.size — byte size
 - def.GetTypeName — type name
 - Set IsSigned := 1 (all floats are signed)

 Step 3: appenddef_string — String type records

 Switch on def.stringtype (from TStringDef):
 - st_shortstring → emit REC_SHORTSTR: TypeID(4) + MaxLength(1) + NameLen(2) + Name
   - def.len gives max length
 - st_ansistring → emit REC_ANSISTR: TypeID(4) + NameLen(2) + Name
 - st_unicodestring, st_widestring → emit REC_UNICODESTR: TypeID(4) + NameLen(2) + Name

 Step 4: appenddef_pointer — Pointer type records

 Emit REC_POINTER: TypeID(4) + TargetTypeID(4) + NameLen(2) + Name

 Extract from TPointerDef:
 - def.pointeddef — target type (get its TypeID via FTypeMapper.GetTypeID)
 - Handle void pointers: use TypeID 0 for target when is_voidpointer(def)

 Step 5: appenddef_array — Array type records

 Emit REC_ARRAY: TypeID(4) + ElementTypeID(4) + Dimensions(1) + IsDynamic(1) + NameLen(2) + Name [+ Bounds for static]

 Extract from TArrayDef:
 - def.elementdef — element type
 - is_dynamic_array(def) — static vs dynamic
 - def.lowrange, def.highrange — bounds for static arrays
 - Bounds are TArrayBound records: LowerBound(8) + UpperBound(8) per dimension

 Step 6: appenddef_record — Record type records

 Emit REC_RECORD: TypeID(4) + FieldCount(4) + TotalSize(4) + NameLen(2) + Name + Fields[]

 Each field is: FieldTypeID(4) + Offset(4) + FieldNameLen(2) + FieldName

 Extract from TRecordDef:
 - def.size — total record size
 - def.objname — record name (may be nil for anonymous)
 - Iterate def.symtable.SymList for TFieldVarSym entries
   - sym.fieldoffset — byte offset
   - sym.vardef — field type (get TypeID)
   - Skip static fields (sp_static in sym.symoptions)

 Note: appendsym_fieldvar is intentionally left as a no-op — fields are emitted inline within the record/class record.

 Step 7: appenddef_object — Class type records

 Emit REC_CLASS: TypeID(4) + ParentTypeID(4) + VMTAddress(8) + InstanceSize(4) + FieldCount(4) + NameLen(2) + Name + Fields[]

 Extract from TObjectDef:
 - Only handle odt_class and odt_object initially
 - def.childof — parent class (TypeID 0 if none)
 - tobjectsymtable(def.symtable).datasize — instance size
 - Iterate def.symtable.SymList for fields (same as records)
 - VMT address: emit as 0 (the linker-resolved VMT label could be added later)

 Step 8: appendprocdef — Function scope records (new override)

 Override appendprocdef to emit REC_FUNCSCOPE: ScopeID(4) + LowPC(8) + HighPC(8) + NameLen(2) + Name

 Extract from TProcDef:
 - def.mangledname — function name (used as both name and symbol reference)
 - def.procstarttai / def.procendtai — need labels for low/high PC
 - ScopeID: use the function's entry address (linker-resolved label)
 - Also call write_symtable_parasyms and write_symtable_syms(localst) to trigger local/param emission

 Key challenge: ScopeID must match between function scope and local var records. Use a simple counter or the address itself. The standalone
 debugger uses the low PC as ScopeID.

 For the embedded format, LowPC and HighPC must be symbol references (labels), not absolute addresses. Emit via tai_const.Create_type_sym.

 For HighPC, insert a label before procendtai (same pattern as DWARF).

 Step 9: appendsym_localvar — Local variable records

 Emit REC_LOCALVAR: TypeID(4) + ScopeID(4) + LocationExpr(1) + NameLen(2) + LocationData(1) + Name

 Extract from TLocalVarSym:
 - sym.vardef — type
 - sym.localloc.reference.offset — RBP-relative stack offset (when sym.localloc.loc is reference-based)
 - sym.owner.defowner — owning procdef (for scope ID)
 - LocationExpr: 1 = RBP-relative (the only mode we support initially)
 - LocationData: the ShortInt stack offset

 Limitation: The current OPDF format uses ShortInt for location data, which limits offsets to -128..127. For deeper stacks, we'll need to
 extend this later. For now, clamp to ShortInt range.

 Step 10: appendsym_paravar — Parameter variable records

 Emit REC_PARAMETER: TypeID(4) + IsVar(1) + IsConst(1) + NameLen(2) + Name

 Extract from TParaVarSym:
 - sym.vardef — type
 - sym.varspez — vs_var, vs_const, vs_out, vs_constref, vs_value

 Note: The current OPDF TDefParameter doesn't include location info. Parameters are identified by their owning function scope. This matches
 the standalone debugger's current capabilities.

 Step 11: appenddef_enum — Full enum type with member names

 Add to ogopdf.pas: New recEnum = 17 with TDefEnum record:

 TDefEnum = packed record
   TypeID: TTypeID;          // 4 bytes
   SizeInBytes: Byte;        // 1 byte (storage size: 1, 2, or 4)
   MemberCount: Cardinal;    // 4 bytes
   NameLen: TNameLen;        // 2 bytes
   // Followed by Name (NameLen bytes)
   // Followed by MemberCount × TEnumMember
 end;

 TEnumMember = packed record
   Value: Int64;             // 8 bytes (supports full range)
   NameLen: TNameLen;        // 2 bytes
   // Followed by Name (NameLen bytes)
 end;


 Add to opdf_io.pas: WriteEnum and ReadEnum methods.

 Implement in dbgopdf.pas: In appenddef_enum:
 - def.size — storage byte size
 - def.GetTypeName — enum type name
 - Iterate def.symtable.SymList for tenumsym entries
 - Each member: tenumsym(entry).value (ordinal) + tenumsym(entry).name (identifier)
 - Filter to def.minval..def.maxval range

 This enables the debugger to display evThird (2) instead of raw 2.

 Step 12: recInterface — New OPDF record type for interfaces

 Add to ogopdf.pas: New recInterface = 16 with TDefInterface record:

 TDefInterface = packed record
   TypeID: TTypeID;          // 4 bytes (ID of this interface type)
   ParentTypeID: TTypeID;    // 4 bytes (parent interface, 0 if IUnknown/root)
   IntfType: Byte;           // 1 byte (0=COM, 1=CORBA, 2=Dispatch)
   GUID: TGUID;              // 16 bytes (interface GUID, zeroed for CORBA)
   MethodCount: Cardinal;    // 4 bytes (number of methods)
   NameLen: TNameLen;        // 2 bytes
   // Followed by Name (NameLen bytes)
   // Followed by MethodCount × TInterfaceMethodDescriptor
 end;

 TInterfaceMethodDescriptor = packed record
   ReturnTypeID: TTypeID;    // 4 bytes (0 for procedure)
   ParamCount: Byte;         // 1 byte
   NameLen: TNameLen;        // 2 bytes
   // Followed by Name (NameLen bytes)
 end;


 This captures method signatures that DWARF currently omits for interfaces.

 Add to opdf_io.pas: WriteInterface and ReadInterface methods matching the format.

 Implement in dbgopdf.pas: In appenddef_object, when def.objecttype is odt_interfacecom, odt_interfacecorba, or odt_dispinterface:
 - Extract GUID from def.iidguid^
 - Distinguish COM/CORBA via def.objecttype
 - Parent interface from def.childof
 - Method list from def.symtable (filter for tprocsym entries)

 Step 13: Interface support in appenddef_object for class declarations

 When emitting recClass records, also emit the list of implemented interfaces. Add ImplIntfCount and ImplIntfTypeID[] fields to the class
 record, or emit separate recInterface references. Decision: extend TDefClass with an ImplIntfCount: Cardinal field after FieldCount, followed
  by ImplIntfCount × TTypeID entries listing the interface TypeIDs.

 Update ogopdf.pas: Extend TDefClass to include implemented interfaces count and IDs after the fields section.

 Implementation Order

 1. Steps 1-4 (constants, float, string, pointer) — independent, simple types
 2. Step 5 (array) — moderate complexity
 3. Step 11 (enum) — new format type, needed before records/classes that contain enum fields
 4. Steps 6-7 (record, class/object) — field iteration, parent/VMT
 5. Steps 8-10 (function scope, locals, params) — interdependent, most complex
 6. Steps 12-13 (interface type, class-implements-interface) — new format additions
